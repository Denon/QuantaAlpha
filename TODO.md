# Walk-Forward TODO

## 待研究（需要搞清楚再落地）

### 1. 市场环境判断
在因子选择中加入对当前市场环境的判断，不同市场环境下使用不同的因子。
- **研究点**：
  - 如何量化市场环境（波动率分档？趋势 vs 震荡？牛熊标记？）
  - 如何将市场状态与因子选择联动（分状态独立排序？还是作为排序权重的协变量？）
  - 过渡期怎么处理（市场状态变化时是平滑过渡还是硬切换？）
- **不增加新配置选项**，复用 `WalkForwardConfig` 已有的字段即可。

### 2. 因子选择跨期稳定性指标
分析相邻 fold 之间选中因子集合的相似度，诊断因子选择是否过拟合。
- **研究点**：
  - Jaccard 相似度是否适合这里？（只关心有无，不关心排名？）
  - 要不要同时看 Top-k 的排序一致性？（比如 Spearman 相关系数）
  - 阈值怎么定？多少算"不稳定"？
- **实现位置**：`_save_result` 或者 `_aggregate_metrics`，不涉及架构改动。

### 3. Fold 并发执行
各 fold 之间无依赖，可以并行跑来缩短总耗时。
- **研究点**：
  - Qlib 的全局状态（`qlib.init`、`R.start`）在并发时会不会互相干扰？
  - 用 `ThreadPoolExecutor` 还是 `ProcessPoolExecutor`？
  - 日志输出怎么管理（多个 fold 同时打日志会混乱）？
- **实现位置**：`WalkForwardBacktestRunner.run()` 里的 fold 迭代循环。

### 4. 折叠置信度加权聚合
聚合指标时按每 fold 选择期的 Rank ICIR 置信度加权，而非简单平均。
- **研究点**：
  - 权重公式：直接 `ICIR` 还是 `ICIR^2` 或 `abs(ICIR)`？
  - 置信度低的 fold 应该被压低还是直接丢弃？
  - 现有用户可能依赖简单平均，需要确保向后兼容或做配置选项。
- **实现位置**：`_aggregate_metrics`

### 5. 数据增量更新
数据来源：https://github.com/chenditc/investment_data
目前是一次性下载全量数据，后续需要研究该仓库是否提供增量更新脚本。
- **研究点**：
  - 该 repo 是否自带增量下载功能？查看 `scripts/` 目录和文档。
  - 如果没有，是每次重新下载全量覆盖（仓库定期更新），还是可以自己写 diff 脚本？
  - Qlib 的数据格式（`.bin` 文件）能否增量追加，还是必须重建？
  - 当前 Qlib 数据目录 `~/.qlib/qlib_data/cn_data/` 的更新频率和磁盘占用。
- **实现位置**：独立脚本或 cron job，不在 backtest 框架内部。
